# Product

## Register

product

## Users
Self-hosting developers and small infrastructure operators who need a compact management console for subscription conversion, API tokens, relay routing, and audit logs.

## Product Purpose
Mexion unifies subscription conversion and API relay management in one local-first control plane. Success means an operator can bootstrap the instance, create a subscription endpoint, issue a limited token, proxy a route, and audit the traffic without SaaS-dashboard clutter.

## Brand Personality
Editorial, rigorous, restrained. The interface should feel like a printed technical journal that happens to operate live infrastructure.

## Anti-references
Avoid generic SaaS dashboards, Ant Design, Element Plus, shadcn defaults, rounded cards, shadows, colorful badges, chart-heavy analytics, emoji, glassmorphism, and marketing-page excess.

## Design Principles
1. Dense but legible: favor ledger rows, clear section numbers, and precise labels over oversized cards.
2. Infrastructure as publication: treat credentials, routes, and logs like annotated plates in a technical journal.
3. Cinnabar is a stamp: red appears sparingly for activation, confirmation, and urgent action only.
4. Every border has a job: use 1px rules and dashed separators to structure information without decoration.
5. Operators first: prioritize copyable endpoints, visible status, keyboard access, and fast verification loops.

## Accessibility & Inclusion
Target WCAG AA contrast, keyboard-operable forms and navigation, visible cinnabar focus borders, reduced decorative motion, and text-first status indicators.
